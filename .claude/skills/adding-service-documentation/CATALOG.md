# Services Catalog (Generated)

The services catalog is no longer hand-curated. Both the listing data and the all-services directory are generated from each service page's frontmatter.

## What Is Generated

| Output file | Generated by | Purpose |
|---|---|---|
| `docs/.vitepress/theme/data/services.json` | `scripts/generate-service-list.mjs` | Consumed by `docs/.vitepress/theme/components/Services/List.vue` to render the searchable listing |
| `docs/services/all.md` | `scripts/generate-services-page.mjs` | Renders the categorized All Services directory page |

Both scripts share `scripts/services-data.mjs`, which:

1. Reads every `docs/services/*.md` (skipping `all.md`, `introduction.md`, `overview.md`)
2. Parses top-level scalar frontmatter keys
3. Resolves the icon by scanning `docs/public/images/services/`
4. Sorts services alphabetically by `name`

**Do not edit `services.json` or `all.md` by hand.** Run the generators instead.

## Running the Generators

```bash
# Both
bun run generate:services

# Individually
bun run generate:services:data   # services.json only
bun run generate:services:page   # all.md only
```

Both also run automatically before `bun run dev`, `bun run build`, and `bun run preview`.

## What Each Field Does

The frontmatter of `docs/services/<slug>.md` directly produces the service entry:

```yaml
---
title: "Service Name"          # → name
description: "..."             # → description
category: "Analytics"          # → category (also the heading in all.md)
icon: "/docs/images/..."       # → icon (optional; auto-resolved when omitted)
disabled: true                 # → disabled (optional)
---
```

The slug is derived from the filename (`my-service.md` → `my-service`). It is also used to generate the URL `/services/my-service`.

## Icon Auto-Resolution

When `icon` is omitted from frontmatter, the resolver in `services-data.mjs` searches `docs/public/images/services/` for a basename matching, in order:

1. `<slug>-logo`
2. `<slug>_logo`
3. `<slug>logo`
4. `<slug>`
5. `<title>-logo`
6. `<title>`

Comparison is case-insensitive and ignores non-alphanumerics. Recognized extensions: `.svg`, `.png`, `.webp`, `.jpg`, `.jpeg`.

If none of those match, it falls back to a fuzzy prefix match (`<slug>logo*` or `<slug>*`), and finally to the first image referenced in the markdown body via `<ZoomableImage src="...">` or `![alt](...)`. If even that fails, the icon is empty.

**Practical guidance:** name the logo `<slug>.svg` or `<slug>-logo.svg` and you'll never need to set `icon` in frontmatter.

## Disabled Services

A service is treated as disabled (hidden from the listing, with `disabled: true` in `services.json`) if **either**:

- frontmatter contains `disabled: true`, **or**
- the markdown body contains one of these patterns (case-insensitive): `SERVICE HIDDEN`, `SERVICE NOT AVAILABLE`, `SERVICE REMOVED FROM COOLIFY`, `SERVICE TEMPORARILY DISABLED`.

Disabled services are still excluded from `all.md` even though they remain accessible by direct URL. See the `disabling-services` skill for the full workflow.

## Categories

The `category` value is used verbatim — both as the filter in the listing and as the `## Heading` in `all.md`. Headings are sorted alphabetically by category. Services are sorted alphabetically by name within each category.

## Slug ↔ Filename Rule

The slug is the filename without `.md`:

- `appsmith.md` → `/services/appsmith`
- `home-assistant.md` → `/services/home-assistant`
- `n8n.md` → `/services/n8n`

Use lowercase kebab-case. Do **not** use camelCase even if the upstream Coolify JSON does (e.g., `denoKV` → `denokv.md`).

## Verifying Your Changes

After adding or editing a service:

```bash
bun run generate:services
```

Then sanity-check:

- `git diff docs/.vitepress/theme/data/services.json` — confirm your service appears with the right fields
- `git diff docs/services/all.md` — confirm the entry shows up under the right category
- `bun run dev` — visit the listing page and the service URL

## What Was Removed (Migration Notes)

Previously, services were registered by appending an entry to a hand-curated array in `List.vue`, and `all.md` was hand-maintained. Both responsibilities are now collapsed into the markdown frontmatter. If you find yourself looking for a `services` array in `List.vue`, you're on the wrong file — `List.vue` now imports the generated `services.json`.
